4. DESAIN & STRUKTUR DATABASE

4.1 Model User

Berikut adalah struktur User Model yang digunakan untuk menyimpan data pengguna:

[SCREENSHOT GAMBAR 4.1 DI SINI]

Gambar 4.1 - Struktur User Model (User.ts)

Penjelasan:
Model User menyimpan informasi pengguna dengan field:
- name: Nama pengguna (required)
- email: Email unik pengguna dengan validasi (required, unique)
- password: Password ter-hash menggunakan bcryptjs (required, minlength 6 chars)
- role: Tipe pengguna (buyer/seller)
- shopName: Nama toko (opsional, hanya untuk seller)

Fitur keamanan:
- Password otomatis di-hash sebelum disimpan (pre-save hook)
- Email di-normalisasi menjadi lowercase
- Timestamps (createdAt, updatedAt) otomatis

---

4.2 Model Product

[SCREENSHOT GAMBAR 4.2 DI SINI]

Gambar 4.2 - Struktur Product Model (Product.ts)

Penjelasan:
Model Product menyimpan data produk dengan field:
- name: Nama produk
- description: Deskripsi produk
- price: Harga produk (min 0)
- category: Kategori produk
- stock: Stok ketersediaan (min 0)
- rating: Rating produk (0-5)
- reviews: Jumlah review
- sellerId: ID penjual dengan index untuk query cepat
- images: Array URL gambar produk

---

4.3 Model Order

[SCREENSHOT GAMBAR 4.3 DI SINI]

Gambar 4.3 - Struktur Order Model (Order.ts)

Penjelasan:
Model Order menyimpan data pesanan pembelian:
- orderId: ID pesanan unik (format: ORD-{timestamp}-{random})
- buyerId, sellerId: Identifikasi pembeli & penjual (indexed)
- items: Array item pesanan dengan detail produk
- totalPrice: Total harga pesanan
- status: Status pesanan (pending/processing/shipped/delivered/cancelled)
- shippingAddress: Alamat pengiriman lengkap
- paymentMethod: Metode pembayaran yang digunakan
- trackingNumber, notes: Informasi pengiriman tambahan

6. IMPLEMENTASI BACKEND API

6.1 Cart API Route

[SCREENSHOT GAMBAR 6.1 DI SINI]

Gambar 6.1 - Cart API Route (GET & POST)

Penjelasan:
Cart API mendukung operasi:
- GET: Mengambil cart pengguna, auto-create jika belum ada
- POST: Menambah item ke cart, auto-increment quantity jika produk sudah ada

Request body untuk POST:
{
  "buyerId": "user123",
  "productId": "prod456",
  "productName": "Wooden Craft",
  "price": 50000,
  "quantity": 1,
  "image": "url-image.jpg"
}
